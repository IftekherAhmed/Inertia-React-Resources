Part 1: Inertia.js with React Basics

Step 1: Setting Up Laravel with Inertia.js and React

Inertia.js acts as a bridge between Laravel (backend) and React (frontend), allowing you to build single-page apps without a separate API. Hereâ€™s how to set it up.

Prerequisites

  • Laravel installed (version 11 recommended).
  • Node.js and npm installed.
  • Basic Laravel knowledge (routes, controllers, views).

Setup Instructions

  1. Create a Laravel Project (if not already done):

    composer create-project laravel/laravel task-manager
cd task-manager

  2. Install Inertia.js: Install the Inertia server-side and client-side packages.

    composer require inertiajs/inertia-laravel
npm install @inertiajs/react react react-dom

  3. Set Up Inertia Middleware:

    • In app/Http/Kernel.php, add the Inertia middleware to the web group:

      protected $middlewareGroups = [
    'web' => [
        // Other middleware...
        \Laravel\Inertia\Middleware\InertiaMiddleware::class,
    ],
];

    • Create a root template (resources/views/app.blade.php):

      <!DOCTYPE html>
<html>
<head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    @viteReactRefresh
    @vite(['resources/js/app.js'])
    @inertiaHead
</head>
<body>
    @inertia
</body>
</html>

  4. Configure React with Vite:

    • Update vite.config.js:

      import { defineConfig } from 'vite';
import laravel from 'laravel-vite-plugin';
import react from '@vitejs/plugin-react';

export default defineConfig({
    plugins: [
        laravel({
            input: ['resources/js/app.js'],
            refresh: true,
        }),
        react(),
    ],
});

    • Create resources/js/app.js:

      import { createRoot } from 'react-dom/client';
import { createInertiaApp } from '@inertiajs/react';

createInertiaApp({
    resolve: (name) => import(`./Pages/${name}.jsx`),
    setup({ el, App, props }) {
        createRoot(el).render(<App {...props} />);
    },
});

  5. Create a Task Model and Migration:

    php artisan make:model Task -m

    • Update the migration (database/migrations/..._create_tasks_table.php):

      Schema::create('tasks', function (Blueprint $table) {
    $table->id();
    $table->string('title');
    $table->text('description')->nullable();
    $table->boolean('completed')->default(false);
    $table->timestamps();
});

    • Run migrations:

      php artisan migrate

  6. Create a Controller:

    php artisan make:controller TaskController

    • Update app/Http/Controllers/TaskController.php:

      namespace App\Http\Controllers;

use Inertia\Inertia;
use Illuminate\Http\Request;

class TaskController extends Controller
{
    public function index()
    {
        $tasks = Task::all();
        return Inertia::render('Tasks/Index', ['tasks' => $tasks]);
    }
}

  7. Set Up Routes:

    • In routes/web.php:

      use App\Http\Controllers\TaskController;

Route::get('/tasks', [TaskController::class, 'index'])->name('tasks.index');

  8. Create a React Component:

    • Create resources/js/Pages/Tasks/Index.jsx:

      import React from 'react';

function Index({ tasks }) {
    return (
        <div>
            <h1>Task Manager</h1>
            <ul>
                {tasks.map((task) => (
                    <li key={task.id}>
                        {task.title} - {task.completed ? 'Done' : 'Pending'}
                    </li>
                ))}
            </ul>
        </div>
    );
}

export default Index;

  9. Run the Application:

    php artisan serve
npm run dev
    • Visit http://localhost:8000/tasks to see the task list.

Explanation: This sets up a Laravel app with Inertia and React. The TaskController fetches tasks from the database and passes them to the Index.jsx component via Inertia. The React component renders the task list dynamically.

1. Advanced Hooks

a. useReducer for Complex State Management

Instead of multiple useState calls, use useReducer to manage the form state in Create.jsx.

  • Update Create.jsx:

    import React, { useReducer } from 'react';
import { Inertia } from '@inertiajs/inertia';

const initialState = { title: '', description: '' };

function reducer(state, action) {
    switch (action.type) {
        case 'UPDATE_FIELD':
            return { ...state, [action.field]: action.value };
        case 'RESET':
            return initialState;
        default:
            return state;
    }
}

function Create() {
    const [form, dispatch] = useReducer(reducer, initialState);

    const handleSubmit = (e) => {
        e.preventDefault();
        Inertia.post('/tasks', form, {
            onSuccess: () => dispatch({ type: 'RESET' }),
        });
    };

    const handleChange = (field) => (e) => {
        dispatch({ type: 'UPDATE_FIELD', field, value: e.target.value });
    };

    return (
        <div>
            <h1>Create Task</h1>
            <form onSubmit={handleSubmit}>
                <div>
                    <label>Title</label>
                    <input type="text" value={form.title} onChange={handleChange('title')} />
                </div>
                <div>
                    <label>Description</label>
                    <textarea value={form.description} onChange={handleChange('description')} />
                </div>
                <button type="submit">Create</button>
            </form>
        </div>
    );
}

export default Create;

Explanation: useReducer centralizes form state updates, making it easier to handle complex logic (e.g., resetting the form after submission).

b. useMemo and useCallback for Performance

Optimize a task list filter in Index.jsx to avoid unnecessary re-renders.

  • Update Index.jsx:

    import React, { useState, useMemo, useCallback } from 'react';
import { Link, usePage } from '@inertiajs/react';

function Index({ tasks }) {
    const { flash } = usePage().props;
    const [filter, setFilter] = useState('all');

    const filteredTasks = useMemo(() => {
        return tasks.filter((task) => {
            if (filter === 'completed') return task.completed;
            if (filter === 'pending') return !task.completed;
            return true;
        });
    }, [tasks, filter]);

    const handleFilterChange = useCallback((newFilter) => {
        setFilter(newFilter);
    }, []);

    return (
        <div>
            <h1>Task Manager</h1>
            {flash.message && <div style={{ color: 'green' }}>{flash.message}</div>}
            <Link href="/tasks/create">Add New Task</Link>
            <div>
                <button onClick={() => handleFilterChange('all')}>All</button>
                <button onClick={() => handleFilterChange('completed')}>Completed</button>
                <button onClick={() => handleFilterChange('pending')}>Pending</button>
            </div>
            <ul>
                {filteredTasks.map((task) => (
                    <li key={task.id}>
                        {task.title} - {task.completed ? 'Done' : 'Pending'}
                    </li>
                ))}
            </ul>
        </div>
    );
}

export default Index;

Explanation: useMemo caches the filtered task list, recomputing only when tasks or filter changes. useCallback memoizes the handleFilterChange function to prevent re-renders of child components.

c.Â Custom Hooks

Create a custom hook to handle form submissions with Inertia.

  • Create resources/js/Hooks/useInertiaForm.js:

    import { useReducer } from 'react';
import { Inertia } from '@inertiajs/inertia';

function formReducer(state, action) {
    switch (action.type) {
        case 'UPDATE_FIELD':
            return { ...state, [action.field]: action.value };
        case 'RESET':
            return action.initialState;
        default:
            return state;
    }
}

export function useInertiaForm(initialState) {
    const [form, dispatch] = useReducer(formReducer, initialState);

    const updateField = (field) => (e) => {
        dispatch({ type: 'UPDATE_FIELD', field, value: e.target.value });
    };

    const submit = (url, method = 'post') => {
        Inertia[method](url, form, {
            onSuccess: () => dispatch({ type: 'RESET', initialState }),
        });
    };

    return { form, updateField, submit };
}

  • Update Create.jsx:

    import React from 'react';
import { useInertiaForm } from '../Hooks/useInertiaForm';

function Create() {
    const { form, updateField, submit } = useInertiaForm({ title: '', description: '' });

    return (
        <div>
            <h1>Create Task</h1>
            <form onSubmit={(e) => { e.preventDefault(); submit('/tasks'); }}>
                <div>
                    <label>Title</label>
                    <input type="text" value={form.title} onChange={updateField('title')} />
                </div>
                <div>
                    <label>Description</label>
                    <textarea value={form.description} onChange={updateField('description')} />
                </div>
                <button type="submit">Create</button>
            </form>
        </div>
    );
}

export default Create;

Explanation: The useInertiaForm hook encapsulates form logic, making it reusable across components.

c.Â Managing Asynchronous Data with Thunks

Fetch tasks asynchronously using a Redux thunk.

  • Update taskSlice.js:

    import { createSlice, createAsyncThunk } from '@reduxjs/toolkit';

export const fetchTasks = createAsyncThunk('tasks/fetchTasks', async () => {
    const response = await fetch('/api/tasks');
    return response.json();
});

const taskSlice = createSlice({
    name: 'tasks',
    initialState: { tasks: [], status: 'idle', error: null },
    reducers: {
        setTasks: (state, action) => {
            state.tasks = action.payload;
        },
    },
    extraReducers: (builder) => {
        builder
            .addCase(fetchTasks.pending, (state) => {
                state.status = 'loading';
            })
            .addCase(fetchTasks.fulfilled, (state, action) => {
                state.status = 'succeeded';
                state.tasks = action.payload;
            })
            .addCase(fetchTasks.rejected, (state, action) => {
                state.status = 'failed';
                state.error = action.error.message;
            });
    },
});

export const { setTasks } = taskSlice.actions;
export default taskSlice.reducer;

  • Create a Laravel API route (routes/api.php):

Route::get('/tasks', [TaskController::class, 'apiIndex'])->name('api.tasks');

                        {task.title} - {task.completed ? 'Done' : 'Pending'}
                    </li>
                ))}
            </ul>
        </div>
    );
}

export default Index;

Explanation: The fetchTasks thunk handles async API calls, updating the Redux store with loading, success, or error states.

c.Â Validation with Yup

Add client-side validation with Yup.

  • Install:

    npm install @hookform/resolvers yup

  • Update Create.jsx:

    import React from 'react';
import { useForm } from 'react-hook-form';
import { yupResolver } from '@hookform/resolvers/yup';
import * as Yup from 'yup';
import { Inertia } from '@inertiajs/inertia';

const schema = Yup.object().shape({
    title: Yup.string().required('Title is required').max(255, 'Title is too long'),
    description: Yup.string().nullable(),
});

function Create() {
    const { register, handleSubmit, formState: { errors }, reset } = useForm({
        resolver: yupResolver(schema),
        defaultValues: { title: '', description: '' },
    });

    const onSubmit = (data) => {
        Inertia.post('/tasks', data, { onSuccess: () => reset() });
    };

    return (
        <div className="max-w-2xl mx-auto p-4">
            <h1 className="text-2xl font-bold">Create Task</h1>
            <form onSubmit={handleSubmit(onSubmit)}>
                <div className="mb-4">
                    <label className="block text-gray-700">Title</label>
                    <input
                        {...register('title')}
                        className="border p-2 w-full"
                    />
                    {errors.title && <p className="text-red-600">{errors.title.message}</p>}
                </div>
                <div className="mb-4">
                    <label className="block text-gray-700">Description</label>
                    <textarea {...register('description')} className="border p-2 w-full" />
                </div>
                <button type="submit" className="bg-blue-500 text-white p-2">
                    Create
                </button>
            </form>
        </div>
    );
}

export default Create;

Explanation: Yup integrates with React Hook Form to provide robust client-side validation, complementing Laravelâ€™s server-side validation.
